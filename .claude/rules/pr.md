GitHub Pull Requestを自動作成します。

## 実行内容

### 1. 現状確認
```bash
git status
git diff --stat
git log --oneline -10
```

### 2. 変更内容の分析
- 変更ファイル一覧
- 追加/削除行数
- 変更の種類（機能追加/バグ修正/リファクタ等）

### 3. PR作成
```bash
gh pr create --title "タイトル" --body "本文"
```

## PRテンプレート

```markdown
## 概要
<!-- 変更の概要を簡潔に -->

## 変更内容
- 変更点1
- 変更点2

## テスト方法
- [ ] 動作確認済み
- [ ] テスト実行済み

## スクリーンショット（UI変更の場合）

## 関連Issue
Closes #
```

## オプション

| オプション | 説明 |
|-----------|------|
| `--draft` | ドラフトPRとして作成 |
| `--base <branch>` | ベースブランチを指定（デフォルト: main） |
| `--reviewer <user>` | レビュアーを指定 |

$ARGUMENTS でベースブランチまたはオプションを指定。

## 自動判定

| 変更内容 | PRタイトルプレフィックス |
|---------|------------------------|
| 新機能 | `feat:` |
| バグ修正 | `fix:` |
| リファクタ | `refactor:` |
| ドキュメント | `docs:` |
| テスト | `test:` |
