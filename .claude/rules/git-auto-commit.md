# Git自動コミット・プッシュルール（絶対遵守）

> ⚠️ **ファイル更新時は必ずGitHubにコミット・プッシュすること**

## 🔴 最重要ルール

| ルール | 内容 |
|--------|------|
| **即時コミット** | ファイル更新後、即座にコミット |
| **即時プッシュ** | コミット後、即座にプッシュ |
| **忘れない** | コンテキスト圧縮後もこのルールを維持 |

## 対象

- CLAUDE.md の更新
- .claude/rules/*.md の追加・編集・削除
- .claude/commands/*.md の追加・編集・削除
- .claude/settings.json の更新
- その他設定ファイルの変更

## コミット手順

```bash
cd D:/extensions/CLAUDE_STTING
git add .
git commit -m "[種類] 変更内容"
git push origin master
```

## コミットプレフィックス

| プレフィックス | 用途 |
|---------------|------|
| `[feat]` | 新規ルール追加 |
| `[fix]` | ルール修正 |
| `[update]` | ルール更新 |
| `[delete]` | ルール削除 |
| `[refactor]` | 整理・リファクタ |

## 禁止事項

- ❌ 更新後にコミットを忘れる
- ❌ コミット後にプッシュを忘れる
- ❌ 「後でまとめてコミット」は禁止
- ❌ コンテキスト圧縮後にこのルールを忘れる

## チェックリスト

ルール編集後：
- [ ] git add .
- [ ] git commit -m "[種類] 変更内容"
- [ ] git push origin master
